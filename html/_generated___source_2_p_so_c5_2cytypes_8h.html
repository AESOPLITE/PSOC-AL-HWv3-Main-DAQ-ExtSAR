Detailed Description

CyTypes provides register access macros and approved types for use in firmware.

Version
6.10
Note
Due to endiannesses of the hardware and some compilers, the register access macros for big endian compilers use some library calls to arrange data the correct way.

Register Access macros and functions perform their operations on an input of the type pointer to void. The arguments passed to it should be pointers to the type associated with the register size. (i.e. a "uint8 *" shouldn't be passed to obtain a 16-bit register value)

Defines the standard return values used in PSoC content. A function is not limited to these return values but can use them when returning standard error values. Return values can be overloaded if documented in the function header. On the 8051 a function can use a larger return type but still use the defined return codes.

Zero is successful, all other values indicate some form of failure. 1 - 0x7F - standard defined values; 0x80 - ... - user or content defined values.
