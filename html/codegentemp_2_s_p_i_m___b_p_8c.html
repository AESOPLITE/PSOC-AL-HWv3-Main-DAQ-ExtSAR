◆ SPIM_BP_ClearRxBuffer()

void SPIM_BP_ClearRxBuffer ( void  )

Function Name: SPIM_BP_ClearRxBuffer

Summary: Clear the RX RAM buffer by setting the read and write pointers both to zero.

Parameters: None.

Return: None.

Global variables: SPIM_BP_rxBufferWrite - used for the account of the bytes which have been written down in the RX software buffer, modified every function call - resets to zero. SPIM_BP_rxBufferRead - used for the account of the bytes which have been read from the RX software buffer, modified every function call - resets to zero.

Theory: Setting the pointers to zero makes the system believe there is no data to read and writing will resume at address 0 overwriting any data that may have remained in the RAM.

Side Effects: Any received data not read from the RAM buffer will be lost when overwritten.

Reentrant: No.

uint8 SPIM_BP_ReadRxData ( void  )

Function Name: SPIM_BP_ReadRxData

Summary: Read the next byte of data received across the SPI.

Parameters: None.

Return: The next byte of data read from the FIFO.

Global variables: SPIM_BP_rxBufferWrite - used for the account of the bytes which have been written down in the RX software buffer. SPIM_BP_rxBufferRead - used for the account of the bytes which have been read from the RX software buffer, modified every function call if RX Software Buffer is used. SPIM_BP_rxBuffer[SPIM_BP_RX_BUFFER_SIZE] - used to store received data.

Side Effects: Will return invalid data if the FIFO is empty. The user should Call GetRxBufferSize() and if it returns a non-zero value then it is safe to call ReadByte() function.

void SPIM_BP_WriteTxData ( uint8  txData)

Function Name: SPIM_BP_WriteTxData

Summary: Write a byte of data to be sent across the SPI.

Parameters: txDataByte: The data value to send across the SPI.

Return: None.

Global variables: SPIM_BP_txBufferWrite - used for the account of the bytes which have been written down in the TX software buffer, modified every function call if TX Software Buffer is used. SPIM_BP_txBufferRead - used for the account of the bytes which have been read from the TX software buffer. SPIM_BP_txBuffer[SPIM_BP_TX_BUFFER_SIZE] - used to store data to sending, modified every function call if TX Software Buffer is used.

Theory: Allows the user to transmit any byte of data in a single transfer.

Side Effects: If this function is called again before the previous byte is finished then the next byte will be appended to the transfer with no time between the byte transfers. Clear Tx status register of the component.

Reentrant: No.
