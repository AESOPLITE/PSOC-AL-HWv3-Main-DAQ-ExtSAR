void UART_Cmd_Sleep ( void  )

Function Name: UART_Cmd_Sleep

Summary: This is the preferred API to prepare the component for sleep. The UART_Cmd_Sleep() API saves the current component state. Then it calls the UART_Cmd_Stop() function and calls UART_Cmd_SaveConfig() to save the hardware configuration. Call the UART_Cmd_Sleep() function before calling the CyPmSleep() or the CyPmHibernate() function.

Parameters: None.

Return: None.

Global Variables: UART_Cmd_backup - modified when non-retention registers are saved.

Reentrant: No.

void UART_Cmd_Wakeup ( void  )

Function Name: UART_Cmd_Wakeup

Summary: This is the preferred API to restore the component to the state when UART_Cmd_Sleep() was called. The UART_Cmd_Wakeup() function calls the UART_Cmd_RestoreConfig() function to restore the configuration. If the component was enabled before the UART_Cmd_Sleep() function was called, the UART_Cmd_Wakeup() function will also re-enable the component.

Parameters: None.

Return: None.

Global Variables: UART_Cmd_backup - used when non-retention registers are restored.

Reentrant: No.
