void UART_LR_Cmd_1_Sleep ( void  )

Function Name: UART_LR_Cmd_1_Sleep

Summary: This is the preferred API to prepare the component for sleep. The UART_LR_Cmd_1_Sleep() API saves the current component state. Then it calls the UART_LR_Cmd_1_Stop() function and calls UART_LR_Cmd_1_SaveConfig() to save the hardware configuration. Call the UART_LR_Cmd_1_Sleep() function before calling the CyPmSleep() or the CyPmHibernate() function.

Parameters: None.

Return: None.

Global Variables: UART_LR_Cmd_1_backup - modified when non-retention registers are saved.

Reentrant: No.

void UART_LR_Cmd_1_Wakeup ( void  )

Function Name: UART_LR_Cmd_1_Wakeup

Summary: This is the preferred API to restore the component to the state when UART_LR_Cmd_1_Sleep() was called. The UART_LR_Cmd_1_Wakeup() function calls the UART_LR_Cmd_1_RestoreConfig() function to restore the configuration. If the component was enabled before the UART_LR_Cmd_1_Sleep() function was called, the UART_LR_Cmd_1_Wakeup() function will also re-enable the component.

Parameters: None.

Return: None.

Global Variables: UART_LR_Cmd_1_backup - used when non-retention registers are restored.

Reentrant: No.
