void UART_LR_Cmd_2_ClearRxBuffer ( void  )

Function Name: UART_LR_Cmd_2_ClearRxBuffer

Summary: Clears the receiver memory buffer and hardware RX FIFO of all received data.

Parameters: None.

Return: None.

Global Variables: UART_LR_Cmd_2_rxBufferWrite - cleared to zero. UART_LR_Cmd_2_rxBufferRead - cleared to zero. UART_LR_Cmd_2_rxBufferLoopDetect - cleared to zero. UART_LR_Cmd_2_rxBufferOverflow - cleared to zero.

Reentrant: No.

Theory: Setting the pointers to zero makes the system believe there is no data to read and writing will resume at address 0 overwriting any data that may have remained in the RAM.

Side Effects: Any received data not read from the RAM or FIFO buffer will be lost.

uint8 UART_LR_Cmd_2_GetRxBufferSize ( void  )

Function Name: UART_LR_Cmd_2_GetRxBufferSize

Summary: Returns the number of received bytes available in the RX buffer.

  • RX software buffer is disabled (RX Buffer Size parameter is equal to 4): returns 0 for empty RX FIFO or 1 for not empty RX FIFO.
  • RX software buffer is enabled: returns the number of bytes available in the RX software buffer. Bytes available in the RX FIFO do not take to account.

Parameters: None.

Return: uint8: Number of bytes in the RX buffer. Return value type depends on RX Buffer Size parameter.

Global Variables: UART_LR_Cmd_2_rxBufferWrite - used to calculate left bytes. UART_LR_Cmd_2_rxBufferRead - used to calculate left bytes. UART_LR_Cmd_2_rxBufferLoopDetect - checked to decide left bytes amount.

Reentrant: No.

Theory: Allows the user to find out how full the RX Buffer is.

uint8 UART_LR_Cmd_2_ReadRxStatus ( void  )

Function Name: UART_LR_Cmd_2_ReadRxStatus

Summary: Returns the current state of the receiver status register and the software buffer overflow status.

Parameters: None.

Return: Current state of the status register.

Side Effect: All status register bits are clear-on-read except UART_LR_Cmd_2_RX_STS_FIFO_NOTEMPTY. UART_LR_Cmd_2_RX_STS_FIFO_NOTEMPTY clears immediately after RX data register read.

Global Variables: UART_LR_Cmd_2_rxBufferOverflow - used to indicate overload condition. It set to one in RX interrupt when there isn't free space in UART_LR_Cmd_2_rxBufferRead to write new data. This condition returned and cleared to zero by this API as an UART_LR_Cmd_2_RX_STS_SOFT_BUFF_OVER bit along with RX Status register bits.

void UART_LR_Cmd_2_SetRxInterruptMode ( uint8  intSrc)

Function Name: UART_LR_Cmd_2_SetRxInterruptMode

Summary: Configures the RX interrupt sources enabled.

Parameters: IntSrc: Bit field containing the RX interrupts to enable. Based on the bit-field arrangement of the status register. This value must be a combination of status register bit-masks shown below: UART_LR_Cmd_2_RX_STS_FIFO_NOTEMPTY Interrupt on byte received. UART_LR_Cmd_2_RX_STS_PAR_ERROR Interrupt on parity error. UART_LR_Cmd_2_RX_STS_STOP_ERROR Interrupt on stop error. UART_LR_Cmd_2_RX_STS_BREAK Interrupt on break. UART_LR_Cmd_2_RX_STS_OVERRUN Interrupt on overrun error. UART_LR_Cmd_2_RX_STS_ADDR_MATCH Interrupt on address match. UART_LR_Cmd_2_RX_STS_MRKSPC Interrupt on address detect.

Return: None.

Theory: Enables the output of specific status bits to the interrupt controller

Summary: This is the preferred method to begin component operation. UART_LR_Cmd_2_Start() sets the initVar variable, calls the UART_LR_Cmd_2_Init() function, and then calls the UART_LR_Cmd_2_Enable() function.

Parameters: None.

The variable is initialized to zero (0u) and set to one (1u) the first time UART_LR_Cmd_2_Start() is called. This allows for component initialization without re-initialization in all subsequent calls to the UART_LR_Cmd_2_Start() routine.
