◆ RTC_Main_Start()

void RTC_Main_Start ( void  )

Function Name: RTC_Main_Start

Summary: Enables RTC component: configures counter, setup interrupts, done all required calculation and starts counter.

Parameters: None.

Return: None.

Global variables: RTC_Main_initVar: global variable is used to indicate initial configuration of this component. The variable is initialized to zero and set to 1 the first time RTC_Main_Start() is called. This allows for component initialization without re-initialization in all subsequent calls to the RTC_Main_Start() routine.

RTC_Main_currentTimeDate, RTC_Main_dstTimeDateStart, RTC_Main_dstTimeDateStop, RTC_Main_dstTimeDateStart, RTC_Main_alarmCfgTimeDate, RTC_Main_statusDateTime, RTC_Main_dstStartStatus, RTC_Main_dstStopStatus, RTC_Main_alarmCurStatus: global variables are modified by the functions called from RTC_Main_Init().

Reentrant: No.

Side Effects: Enables for the one pulse per second (for the RTC component) and Central Time Wheel (for the Sleep Timer component) signals to wake up device from the low power (Sleep and Alternate Active) modes and leaves them enabled.

The Power Manager API has the higher priority on resource usage: it is NOT guaranteed that the Sleep Timer's configuration will be the same on exit from the Power Manager APIs as on the entry. To prevent this use the Sleep Timer's Sleep() - to save configuration and stop the component and Wakeup() function to restore configuration and enable the component.

The Sleep Timer and Real Time Clock (RTC) components could be configured as a wake up source from the low power modes only both at once.
