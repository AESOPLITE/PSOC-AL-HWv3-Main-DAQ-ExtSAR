Function Name: I2C_RTC_MasterReadBuf

Summary: Automatically writes an entire buffer of data to a slave device. Once the data transfer is initiated by this function, further data transfer is handled by the included ISR in byte by byte mode.

Parameters: slaveAddr: 7-bit slave address. xferData: Pointer to buffer where to put data from slave. cnt: Size of buffer to read. mode: Transfer mode defines: start or restart condition generation at begin of the transfer and complete the transfer or halt before generating a stop.

Return: Status error - Zero means no errors.

Side Effects: The included ISR will start a transfer after start or restart condition is generated.

Global variables: I2C_RTC_mstrStatus - The global variable used to store a current status of the I2C Master. I2C_RTC_state - The global variable used to store a current state of the software FSM. I2C_RTC_mstrControl - The global variable used to control the master end of a transaction with or without Stop generation. I2C_RTC_mstrRdBufPtr - The global variable used to store a pointer to the master write buffer. I2C_RTC_mstrRdBufIndex - The global variable used to store a current index within the master write buffer. I2C_RTC_mstrRdBufSize - The global variable used to store a master write buffer size.

Reentrant: No.

uint8 I2C_RTC_MasterReadByte ( uint8  acknNak)

Function Name: I2C_RTC_MasterReadByte

Summary: Reads one byte from a slave and ACK or NACK the transfer. A valid Start or ReStart condition must be generated before this call this function. Function do nothing if Start or Restart condition was failed before call this function.

Parameters: acknNack: Zero, response with NACK, if non-zero response with ACK.

Return: Byte read from slave.

Side Effects: This function is entered without a "byte complete" bit set in the I2C_CSR register. It does not exit until it is set.

Global variables: I2C_RTC_state - The global variable used to store a current state of the software FSM.

Reentrant: No.
